SHIELD STRATEGIC ANALYSIS TEAM

数据说明
nodes.csv / edges.csv / hero-network.csv
内容：漫威漫画官方原始数据集，包含6000余名英雄及16.5万条原始社交连线。
作为项目原始证据保留。

cleaned_nodes.csv / cleaned_edges.csv / cleaned_hero_network.csv
内容：去噪后的标准化数据集。
处理方法：通过 Pandas 脚本统一字段格式，剔除漫画刊物节点，仅保留英雄间的对等社交关系。

hero_master_final.csv
内容：超人类核心数据库。
扩充内容：在原始数据基础上，通过 DeepSeek API 扩充了中文译名、性别、能力类型、风险评估分数、社交风格标签、MCU电影参与度及核心导师等维度。

hero_tactical_v3.csv
内容：战术动作定义库。包含 40 位核心领袖在“科技、外交、武力”三个维度的战术权重。

rpg_logic_v3.json
内容：因果逻辑汇总。
技术分析：使用 PageRank 算法 预演算了 120 种平行宇宙结局。记录了每个战术选择引发的权力洗牌结果、Winner/Loser 变动以及全网连通度损毁数值。

neighborhood_index.json
内容：社交圈预处理索引。
技术：基于哈希映射预存了每个英雄的核心关系碎片。

shield_index.html：系统入口。

dashboard.html / dossier.html / sim.html / game.html
内容：各模块的 UI 模板。
技术：采用 HTML5/CSS3 ，预留了图表与数据的注入占位符。

build_*.py (共4个)：数据注入引擎。利用 Python 字符串处理技术将演算结果动态填入模板。

rendered.html (共4个)：系统最终运行实体，包含了所有动态图表与游戏逻辑。

技术与分析方法总结
社会网络分析 (SNA)：
PageRank 算法：用于评估社交网络中的权力分配，并模拟节点变动后的影响力漂移。
LCC (最大连通分量)：用于量化漫威宇宙的韧性，计算系统在打击下的损毁率。
Small-World 效应分析：验证全宇宙 3.2 步的平均关联距离。
大语言模型 (LLM) 增强：
利用 DeepSeek 模型对非结构化文本进行语义提取，构建补充数据库。
网页交互与游戏技术：
前端逻辑引擎：在浏览器端内置 BFS (广度优先搜索) 算法，实现生存演习中的实时损毁计算。
坐标空间匹配：利用欧几里得距离算法，计算玩家与英雄的匹配度。
数据岛技术：通过 JSON 注入实现前后端数据通信。

系统操作指南
修改 UI/文字：编辑对应的 .html 模板文件（如 game.html）。
修改数据指标：编辑对应的 hero_tactical_v3.csv 或在 Jupyter 中重新演算。
同步生效：每次完成修改后，必须运行对应的 build_*.py 脚本，系统会自动更新 _rendered.html 文件。

英雄照片添加
存放路径：所有图片放入 ./pic/heroes/ 文件夹。
命名规范：必须使用英雄的英文主名，全小写，空格换下划线。
示例：IRON MAN -> iron_man.jpg；CAPTAIN AMERICA -> captain_america.jpg。
格式要求：建议使用 .jpg 或 .png，比例 3:4。
技术机制：系统具备 onerror 容错。若未找到图片，将自动显示 ./pic/shield_logo.webp 以保证视觉完整。
